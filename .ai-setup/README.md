# AI / IDE Setup — CareNet 2

## What Is This?

CareNet 2 has a complete multi-IDE agentic coding setup. The files in `.github/` are the **canonical Copilot source**. Other IDEs read from their own paths — this file documents where everything lives per IDE.

The project also uses a **shared source-of-truth pack** at:
```
C:\Users\callz\OneDrive\Documents\My Projects\SynologyDrive\Websites\Rules&Skills\packs\carenet-2
```
Run `npm run ai:sync` from the project root to regenerate all IDE rule files from that pack.

> **Do not hand-edit generated files.** Edit the pack, then run `npm run ai:sync`.
> Exception: `.github/agents/`, `.github/skills/`, `.github/prompts/`, and `.github/instructions/`
> are Copilot-specific and managed here directly — they are not generated by the sync script yet.

## VS Code + GitHub Copilot

**Always-on instructions:** `.github/copilot-instructions.md` — loaded automatically every session.

**Scoped instructions** in `.github/instructions/` apply automatically to matching file paths:
- `frontend.instructions.md` → `src/frontend/**`
- `backend-services.instructions.md` → `src/backend/**`
- `migrations.instructions.md` → `supabase/migrations/**`
- `tests.instructions.md` → `e2e/**`, `**/*.test.ts`, `**/*.spec.ts`
- `locales.instructions.md` → `src/locales/**`

**Custom agents** in `.github/agents/` — select from the agent picker (@ syntax) in Copilot Chat:
- `@feature-builder` — builds features end-to-end
- `@feature-planner` — writes specs before building
- `@code-reviewer` — read-only review agent
- `@test-runner` — writes and runs Vitest + Playwright tests
- `@migration-author` — writes Supabase SQL migrations

**Reusable prompts** in `.github/prompts/` — select from the prompt picker (/ slash commands):
- `/new-feature` — start a feature from a brief
- `/code-review` — review code against CareNet 2 constraints
- `/new-migration` — write an idempotent Supabase migration
- `/write-tests` — write Vitest + Playwright tests
- `/fix-bug` — diagnose and fix a bug
- `/refactor` — refactor without changing behavior

**Agent Skills** in `.github/skills/` — Copilot loads these automatically when relevant:
- `fullstack-carenet`, `supabase-dexie-sync`, `tailwind-radix-motion`
- `i18next-bangla`, `capacitor-mobile`, `vitest-playwright`
- `code-review-carenet`, `spec-writing-carenet`

## Keeping Everything In Sync

**Source of truth for rules:** `Rules&Skills/packs/carenet-2` (sibling repo)
**Sync command:** `npm run ai:sync` (runs `scripts/sync-ai-context.mjs`)

**What the sync script regenerates:**
- `.cursor/rules/` and `.cursor/skills/`
- `.github/copilot-instructions.md`
- `.rules`, `rules.md`, `.windsurfrules`
- `.kilocode/rules/`, `.kilocode/rules.md`, `.kilocode/skills/`
- `.agents/rules/`, `.agents/skills/`
- `.agent/rules/`

**What the sync script does NOT touch (Copilot-specific, managed here):**
- `.github/agents/*.agent.md`
- `.github/skills/*/SKILL.md`
- `.github/prompts/*.prompt.md`
- `.github/instructions/*.instructions.md`

After updating these Copilot-specific files, also manually mirror any rule changes to the pack source so the next `ai:sync` doesn't overwrite `copilot-instructions.md` with a stale copy. See `docs/ai-rules-sync.md`.

## Adding a New Skill

1. Create `.github/skills/<skill-name>/SKILL.md` with YAML frontmatter (`name`, `description`) and Markdown instructions.
2. Also create `.agents/skills/<skill-name>/SKILL.md` with the same content (used by non-Copilot agent runners).
3. Register both in this README under the skills table.
4. Add the skill to the pack in `Rules&Skills/packs/carenet-2/skills/` so future syncs include it.

---

## Adding a New Agent

1. Create `.github/agents/<name>.agent.md` with YAML frontmatter (`name`, `description`, `tools`, `handoffs`).
2. Register it in `.github/copilot-instructions.md` under the Agents table.
3. Register it in this README.
