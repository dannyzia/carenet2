# Plan: Wire up Admin `mfaRequired` Toggle to Control MFA

## Current State — Three Gaps

| # | Gap | Detail |
|---|-----|--------|
| 1 | **Toggle doesn't persist** | `AdminSettingsPage.tsx:31` — "Save" button shows toast but never writes to any backend or localStorage |
| 2 | **No settings store** | No context/store exists for non-admin code to read `mfaRequired`. Each consumer calls `adminService.getSettingsData()` independently |
| 3 | **Login never checks the setting** | `AuthContext.tsx` login flow is hardcoded — mock always demands MFA, Supabase never enforces it |

Additionally: the Supabase branch of `adminService.getSettingsData()` (`admin.service.ts:672-675`) returns a hardcoded stub that doesn't include `mfaRequired`. It always falls back to mock/empty defaults.

## Approach: localStorage-backed settings with runtime reads

Use `localStorage` as the persistence layer for admin settings (works for both mock and Supabase mode). AuthContext reads `mfaRequired` from it during login. Admin page reads/writes it.

This avoids creating a Supabase migration for a settings table and works identically in demo/offline mode.

## Changes (5 files, 1 new)

### 1. NEW `src/frontend/auth/mfaConfig.ts` — shared MFA state

```ts
const SETTINGS_KEY = "carenet-admin-settings";

/** Default settings (used when nothing is persisted yet) */
const DEFAULTS = { mfaRequired: false };

function loadSettings(): Record<string, any> {
  try {
    const raw = localStorage.getItem(SETTINGS_KEY);
    return raw ? { ...DEFAULTS, ...JSON.parse(raw) } : { ...DEFAULTS };
  } catch {
    return { ...DEFAULTS };
  }
}

function saveSettings(settings: Record<string, any>) {
  localStorage.setItem(SETTINGS_KEY, JSON.stringify(settings));
}

/** Check if MFA is required — reads from persisted admin settings */
export function isMfaRequired(): boolean {
  return !!loadSettings().mfaRequired;
}

/** Persist the full settings object (called by admin settings page save) */
export function persistAdminSettings(settings: Record<string, any>) {
  saveSettings(settings);
}

/** Load all persisted admin settings (used by admin settings page to init) */
export function loadAdminSettings(): Record<string, any> | null {
  const raw = localStorage.getItem(SETTINGS_KEY);
  return raw ? JSON.parse(raw) : null;
}
```

## How it works end-to-end

1. **Default state:** `mfaRequired: false` — all logins skip MFA
2. **Admin enables MFA:** Admin goes to Settings → Security → toggles "Require 2FA" ON → clicks Save → `persistAdminSettings()` writes to localStorage
3. **Next login:** `AuthContext.login()` calls `isMfaRequired()` → reads `true` from localStorage → MFA verify flow activates
4. **Admin disables MFA:** Toggle OFF → Save → `mfaRequired: false` in localStorage → MFA skipped again
