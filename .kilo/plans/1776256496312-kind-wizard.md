### Implementation Pattern (matching caregiver-full.spec.ts)

```ts
import { test, expect } from "@playwright/test";
import {
  loginAsReal,
  goto,
  mainLandmark,
  assertTitle,
  assertToast,
  captureConsoleErrors,
} from "./helpers";

test.use({ launchOptions: { slowMo: 500 } });

test.describe("Guardian – Complete Role Workflows", () => {
  test.describe.configure({ timeout: 180_000, mode: "serial" });

  test.beforeEach(async ({ page }) => {
    await loginAsReal(page, "guardian");
    await expect(page).toHaveURL(/\/guardian\/dashboard|\/dashboard/);
  });

  // ... 35 tests as above
});
```

### Selectors

All assertions use the same helper pattern as the existing specs:
- `mainLandmark(page)` — scopes to `<main>` (avoids sidebar/nav duplicate headings)
- `goto(page, path)` — navigates and waits for `<main>` visibility
- `captureConsoleErrors(page)` — captures JS errors during test
- `assertToast(page, /pattern/)` — verifies Sonner toast appears
- `getByRole`, `getByText`, `getByPlaceholder` — preferred over CSS selectors
