# Plan — ROLE_SIDEBAR_PERSONA_BUTTON

**Spec:** `Do_Not_Commit/ROLE_SIDEBAR_PERSONA_BUTTON.md`
**Goal:** Split the existing sidebar into (a) a new **RoleSidebar** containing only role-specific nav, opened via a new **Persona** button that replaces Search in `BottomNav`, and (b) a **Legacy Menu** (existing sidebar minus role nav) retaining Browse / Support / App Controls, still opened by the Menu tab.

---

## 1. Key Discovery (refines the spec)

The spec suggests *"filter out `sectionKey: browse / supportInfo` from `getRoleNavSections`"*. This is **inaccurate**:

- `getRoleNavSections(t)` in `AuthenticatedLayout.tsx` (lines 70–337) returns **only role-specific sections** (main, tools, finance, health, growth, management, etc.). None of its entries use `sectionKey: "browse"` or `"supportInfo"`.
- **Browse** (Marketplace / Agencies / Shop) and **Support** (Help Center / Ticket / Refund / Privacy / Terms) are **hardcoded inline blocks** at lines 667–740 of `AuthenticatedLayout.tsx`.

**Consequence:** The split is cleaner than the spec implies. No filter predicate is needed.
- RoleSidebar ← render `getRoleNavSections(t)[currentRole]` as-is.
- Legacy Menu ← keep the two hardcoded Collapsible blocks + App Controls block untouched.

## 3. Component design: `RoleSidebar.tsx`

**Location:** `src/frontend/components/shell/RoleSidebar.tsx`

**Props:**
```ts
interface RoleSidebarProps {
  isOpen: boolean;
  onClose: () => void;
}
```

**Internally uses:**
- `useAuth()` → `user`, `logout` (logout optional; spec keeps it in Legacy Menu, so omit here)
- `useTranslation("common")`
- `useLocation()` for active-link highlighting
- `useUnreadCounts(currentRole)` for messages/billing/activation badges (preserve current badge behavior)
- `usePendingProofCount`, `usePendingActivationCount` (preserve)
- `getRoleNavSections(t)[currentRole]` — reused verbatim from `AuthenticatedLayout` (export it from there, or co-locate in a shared module — see §4)

**Layout (mobile + desktop overlay):**
- Container: `fixed left-0 z-50 w-64` with slide transform, `top: calc(var(--cn-header-height) + env(safe-area-inset-top))`, `bottom: calc(var(--cn-bottom-nav-height) + env(safe-area-inset-bottom))`. Same positioning as the current aside.
- **Header block:** avatar circle (role gradient + initial OR `user.avatarUrl`), `user.name`, `t("roles." + role)`, close X button (mobile only, `md:hidden`).
- **Body:** scroll region; map `navSections` → section title (t("sidebar.section." + key)) + link list. Reuse the exact link styling, Collapsible pattern, and badge logic from `AuthenticatedLayout` lines 572–665.
- **Each Link onClick:** call `onClose()` to close on navigate (matches existing `setSidebarOpen(false)` pattern).

**Backdrop:** sibling overlay `fixed inset-0 bg-black/40 z-40 md:hidden` (mobile) plus optional `hidden md:block` variant if we want desktop backdrop. Recommend: keep backdrop on all breakpoints for consistency (confirm with user — see Question 1).

**Keyboard / a11y:**
- `role="dialog"`, `aria-modal="true"`, `aria-label={t("a11y.roleSidebarNav")}`.
- `useEffect` binds `Escape` → `onClose()` while `isOpen`.
- On open, focus first link. On close, restore focus to `document.activeElement` captured at open time (store in ref).
- Simple focus trap via `keydown` handler cycling Tab/Shift+Tab within the dialog.
- Close X button: `aria-label={t("a11y.closeRoleSidebar")}`.

**Performance:** wrap filtered section computation in `useMemo([currentRole, t])`. No other expensive work.

## 4. Shared utility: `getRoleNavSections`

Currently lives inside `AuthenticatedLayout.tsx`. RoleSidebar needs it too.

**Recommendation:** extract to a new file `src/frontend/components/shell/roleNavSections.ts` exporting:

Both `AuthenticatedLayout.tsx` and `RoleSidebar.tsx` import from the new module. This avoids duplication and keeps the refactor additive.

## 6. `BottomNav.tsx` changes

**Imports:** replace `Search` with `User` from `lucide-react`.

**Props:** add optional `isRoleSidebarOpen?: boolean`.

**Tab array:** replace the `search` entry with:
```ts
{ key: "persona", labelKey: "persona", icon: User, action: "persona", to: "" }
```
Remove the `searchPath` computation block entirely.

**New `action: "persona"` branch** (mirrors `menu` branch):
- Compute `const personaEnabled = !!user?.activeRole;`
- If disabled: render the button with `aria-disabled="true"`, `opacity: 0.4`, `pointerEvents: "none"`, `aria-label={t("a11y.personaMenuDisabled")}`. No click handler.
- If enabled: on click → `window.dispatchEvent(new CustomEvent("toggle-rolesidebar"))`. `aria-label={t("a11y.personaMenu")}`.
- If `isRoleSidebarOpen && personaEnabled`: render the top active-indicator bar and colored icon/label (same visual treatment as the Link-tab `isActive` case).

**i18n:** label comes from `t("bottomNav.persona")`.

## 9. Risks & edge cases

1. **`getRoleNavSections` export refactor** could touch many imports — but it currently has **no external importers** (it is defined inline). Extraction is safe and additive.
2. **Active-path highlighting:** both sidebars can show active links if their paths overlap. Currently role and browse paths don't overlap, so no conflict.
3. **Sidebar state race:** if a user taps Menu then Persona quickly, both could be open. Acceptable (two separate overlays), but recommend closing the other when one opens — see Question 2.
4. **Desktop behavior:** spec says Legacy Menu stays persistent. The current `<aside className="... md:static ...">` behavior is preserved. RoleSidebar appears as a mobile-style overlay even on desktop (with backdrop). Confirm desktop UX (Question 1).
5. **`roleNavLinks` flattened map** is re-exported from the new module so role-detection logic in `AuthenticatedLayout` (used to guard `shouldRedirectToOwnDashboard`) keeps working unchanged.
6. **Focus trap implementation** — small hand-rolled trap is acceptable; no new dep needed. Radix Dialog would also work but would change the DOM shape; hand-rolled keeps diff small.
7. **Capacitor / native:** no native APIs touched; still safe to run `npm run build && npx cap sync` as part of release but **not required for this spec.**
