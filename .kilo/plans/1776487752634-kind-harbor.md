# SIDEBAR_LAYOUT_001 — Reposition Sidebar Between Top Bar and Bottom Nav

## 1. IMPACT ANALYSIS

**Affected modules:**
- `src/frontend/components/shell/AuthenticatedLayout.tsx` — primary target, structural restructure
- `src/frontend/components/navigation/PublicNavBar.tsx` — mobile drawer `top` offset fix
- `src/styles/theme.css` — add CSS variable for authenticated header total height

**Risk level:** Medium
- Restructuring AuthenticatedLayout's DOM order affects ALL authenticated pages
- Header extraction changes desktop layout from side-by-side to stacked
- Mobile sidebar positioning must remain pixel-perfect

**Breaking change assessment:** No API/backend changes. Visual layout change on desktop (header now spans full width above sidebar instead of beside it). All existing functionality preserved.

**Pages affected by AuthenticatedLayout change:**
- All profile pages (caregiver, guardian, patient, public profiles)
- All admin pages (users, user inspector, verifications, etc.)
- All agency, shop, moderator pages
- All shared authenticated routes (settings, wallet, billing, contracts)

**Pages NOT affected:**
- Public pages (login, register, role-selection, marketplace) — already have proper three-tier layout via PublicLayout

---

## 2. ARCHITECTURE DECISION

**Chosen approach:** Extract the header from the right flex column and place it above the sidebar+content flex row as a full-width element. Add safe-area-inset-top padding to match PublicNavBar behavior.

**Alternatives considered:**
1. Keep side-by-side but add safe-area — rejected per user request for full-width header
2. Use CSS Grid instead of Flexbox — rejected to minimize diff, current flex approach works
3. Use the shadcn/ui Sidebar primitives — rejected, AuthenticatedLayout has its own hand-rolled sidebar

**Why this approach:**
- Minimal DOM restructuring (move one element up, adjust a few classes)
- Matches the PublicLayout three-tier pattern (PublicNavBar → content → BottomNav)
- Uses existing CSS variables (`--cn-header-height`) and utility classes (`cn-safe-top`)
- Safe-area handling follows established pattern from PublicNavBar

## 5. FRONTEND DESIGN

### 5.1 `AuthenticatedLayout.tsx` — Structural Restructure

**Current DOM order (simplified):**
```
<div min-h-screen flex flex-col>
  <OfflineIndicator />
  <RetryOverlay />
  <NotificationPermissionPrompt />
  <div flex flex-1 min-h-0>              ← flex row
    <aside fixed/static>                 ← sidebar
    <div flex-1 flex flex-col>           ← main column
      <header sticky top-0 z-30 h-14>   ← header (inside column)
      <main flex-1 cn-bottom-safe>      ← page content
    </div>
  </div>
  <BottomNav fixed bottom-0>
</div>
```

**Target DOM order:**
```
<div min-h-screen flex flex-col>
  <OfflineIndicator />
  <RetryOverlay />
  <NotificationPermissionPrompt />
  <header sticky top-0 z-30 h-14 pt-[env(safe-area-inset-top,0px)]>  ← FULL-WIDTH, above flex row
  <div flex flex-1 min-h-0>              ← flex row
    <aside static/fixed>                 ← sidebar
    <div flex-1 flex flex-col>           ← main column (NO header)
      <main flex-1 cn-bottom-safe>      ← page content
    </div>
  </div>
  <BottomNav fixed bottom-0>
</div>
```

**Specific changes to AuthenticatedLayout.tsx:**

1. **Extract `<header>` from inside `<div className="flex-1 flex flex-col min-w-0">` and place it as a sibling BEFORE `<div className="flex flex-1 min-h-0">`**
   - Line ~682-752: Cut the entire `<header>` block
   - Paste it between `<NotificationPermissionPrompt />` and `<div className="flex flex-1 min-h-0">`

2. **Add safe-area-inset-top to header:**
   - Change: `<header className="sticky top-0 z-30 border-b"`
   - To: `<header className="sticky top-0 z-30 border-b pt-[env(safe-area-inset-top,0px)]"`

3. **Update mobile sidebar `top` position:**
   - Change: `className="fixed top-14 left-0 z-50 ..."`
   - To: Remove `top-14` from className, add inline style:
     ```tsx
     style={{
       top: "calc(3.5rem + env(safe-area-inset-top, 0px))",
       // ...existing styles
     }}
     ```
   - This makes the sidebar start below the header including safe-area padding.

4. **Update desktop sidebar height constraint:**
   - Current: `md:static md:h-auto md:max-h-screen md:min-h-0 md:self-stretch md:translate-x-0`
   - Change `md:max-h-screen` to remove it (or set `md:max-h-none`), since the parent flex row now constrains height automatically.
   - The sidebar should use `md:overflow-y-auto` for internal scrolling when content is tall.
   - Change: `md:static md:h-auto md:max-h-none md:min-h-0 md:self-stretch md:translate-x-0`

5. **Remove header border-b duplication:**
   - The header already has `border-b` class. No change needed.

6. **Update `cn-mobile-header-offset` usage (if any pages reference it):**
   - The CSS variable `--cn-mobile-header-height` is `3.5rem`. This may need updating to include safe-area on native, but since it's used via CSS calc elsewhere, it should be fine as-is.

### 5.2 `PublicNavBar.tsx` — Mobile Drawer Top Fix

**Current:** `<aside className="fixed left-0 top-14 z-[70] ...">`
**Problem:** `top-14` (56px) doesn't account for `pt-[env(safe-area-inset-top,0px)]` on the header, causing the drawer to overlap the bottom portion of the header on notched devices.

**Fix:** Remove `top-14` from className, add inline style:
```tsx
style={{
  top: "calc(3.5rem + env(safe-area-inset-top, 0px))",
  // ...existing styles
}}
```

### 5.3 `src/styles/theme.css` — Optional Enhancement

**Add a CSS variable for authenticated header total height (safe-area aware):**
```css
--cn-auth-header-height: calc(3.5rem + env(safe-area-inset-top, 0px));
```

This can be referenced by the sidebar and other components. However, since the changes above use inline styles with `calc()`, this is optional and can be deferred.

## 7. EDGE CASES & FAILURE HANDLING

| Scenario | Expected Behavior |
|----------|-------------------|
| iOS device with notch | Safe-area-inset-top pushes header down, sidebar starts below it |
| Android gesture navigation | Safe-area-inset-bottom already handled in BottomNav, no change |
| Desktop browser (no safe-area) | `env(safe-area-inset-top, 0px)` resolves to 0, layout unchanged |
| Small viewport height | Sidebar internal scroll region handles overflow |
| Sidebar open while keyboard visible | Existing behavior preserved (no change to keyboard handling) |
| Offline banner visible | OfflineIndicator sits above header, pushes everything down |
| Right-to-left (RTL) layout | Sidebar uses `left-0` which may need `start-0` for RTL; existing issue, not in scope |
